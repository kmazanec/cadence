## Iteration risks

- SHOWSTOPPER (F-07 AC#2): all 18 bilateral_pair_id values in data/exercises.json are dangling — verified 0 of 18 resolve, so repo.bilateral_pair() returns None for every catalogue entry. The auto-pairing build path and the 'bilateral_pair returns partner' unit test are unreachable via real data. Mitigation is the gating pre-build decision: exercise the path via a synthetic in-memory fixture repo (recommended) rather than mutating the shipped dataset; make any real-dataset AC#2 assertion conditional with a recorded rationale.
- F-07 safety (ADR-009): the exclusion filter MUST be applied at BOTH layers — pre-filter in _execute_search (LLM never sees contraindicated ids) AND in validate_workout (catches an LLM-invented/echoed contraindicated id). Applying at only one layer defeats the critical-path invariant. validate_workout gains injuries as a defaulted-None 3rd arg so existing 2-arg call sites keep working.
- F-07 pairing safety: an auto-included bilateral partner may itself be contraindicated (e.g. a knee-loading left/right pair under a knee injury). build_workout MUST re-check the partner against contraindicated_ids and skip it (exclusion wins over pairing), AND dedupe against already-selected ids. Add an explicit test for the contraindicated-partner case.
- F-08 accumulator doubling: with add_messages + MemorySaver, re-invoking the thread while also re-seeding messages would double-append. The frozen rule: keep api/chat.py passing messages=[] (append-only, does not clear) and thread prior context into subgraph inputs WITHOUT re-adding the current HumanMessage twice. Getting prior messages into the subgraph input is the actual work; the risk is doing it via state re-seed instead of checkpoint reuse.
- F-09 schema-aware fake seam (ADR-018 lesson, in memory): instrumenting get_model must NOT change its signature or BaseChatModel return type, or bind_tools/structured-output call sites and the router-dispatch fake break. Wrap, don't replace. Secrets redaction is mandatory — never log the API_KEY_ENV value or a SecretStr; the optional vendor tracer must be a no-op when its key is absent (full suite runs identically without it).
- Over-exclusion recovery (F-07 AC#3) overlaps ADR-018 critical-path #3 (output-gate empty-result recovery, owned elsewhere). Confirm ownership so the over-broad-injury graceful-gap test isn't double-implemented; verify the existing empty path (hub.py:218-229) already returns the honest 'couldn't build a workout' reply with zero contraindicated padding.
- Reason visibility: exclusion/pairing decisions happen inside the subgraph but reasons are emitted in the hub boundary node. Frozen approach: boundary node recomputes contraindicated_ids(injuries) (deterministic, cheap) for 'excluded' reasons and inspects the returned workout for auto-added partners for 'added' reasons — avoids growing GeneratorState. If a partner-source mapping proves not reconstructable from the workout alone, that is the only justification to add a lean field to GeneratorState; default to recompute.
